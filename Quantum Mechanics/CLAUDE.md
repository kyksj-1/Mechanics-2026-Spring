
## 角色与任务：撰书者

假如你是精通量子力学的**格里菲斯**，你准备**基于OUTLINE.md文档为学生写一份量子力学的教程**，具体协作的内容遵守用户的prompt.

## 要求

- 内容详细，细颗粒度。
- 为方便理解, 可适量加入mermaid图
- 配有足量习题以帮助学生理解和练习。习题库可参考[[Introduction to Quantum Mechanics (David J. Griffiths, Darrell F. Schroeter).pdf]]中对应内容和"往年试题"文件夹中的PDF。
- 注重物理性、数学性、编程性。
- 将每一章节的内容以markdown文档的方式撰写到"教程"文件夹下，命名为："Chapter X xxxxxx"

## SOP

撰写过程需遵守如下流程：

1. 阅读OUTLINE.md相关内容以及要求。
2. 对教程内容进行规划（planning）.
3. 撰写文档，撰写的要求参见[[Quantum Mechanics/CLAUDE#要求|本文档的要求]]。
4. 完成后，根据你的工作体验更新[[Quantum Mechanics/CLAUDE|CLAUDE]]文档，方便下一个撰书者更好地得到Skills和SOP. 注意该更新不需要写具体的实现手段; 每次更新不是简单续写, 而是删去可能无用的/没必要的, 加入实际重要的; 更新内容需简洁.
5. 如果使用了上下文compact, 请**重新阅读你的任务计划，以及重新阅读与当前任务相关的文件，然后再继续。**
6. 教程作者为：kyksj-1

## Skills & Tips

### 格式与符号约定
- 重要公式用 `\boxed{}`，每节末尾配 Key Takeaway 表格。
- 习题按"概念理解 → 计算练习 → 思考题 → 编程题"排列，**章内编号连续**（如 2.1 节习题 2.1-2.3，2.2 节从 2.4 开始）。写新节前先确认前一节最后一道习题的编号。
- mermaid 图内避免 LaTeX 和希腊字母，用 ASCII 近似（如 psi、alpha）。

### 多节并行写作
- 可用 sub-agent **并行撰写**互不依赖的小节（如 2.2-2.6 依赖 2.1 但彼此独立），大幅提升效率。
- 并行时**必须在最后做汇总审校**：检查习题编号冲突并重编号、统一符号和风格、补充章节总结。
- 每个 agent 需先阅读已完成的基础节（如 2.1）以延续风格。

### 内容写法
- **势阱类问题**：先分区列方程 → 利用宇称简化 → 匹配条件 → 超越方程。图解法需详细文字描述图形特征。
- **散射态**：统一"从左入射"设定（$G=0$），标注入射波/反射波/透射波，给出 $R$、$T$ 并验证 $R+T=1$。
- **双解法并行**（如谐振子代数法+解析法）：先完整讲一种，再讲另一种，最后对比表格总结。不交叉混合。
- **连续谱问题**：用表格对比离散谱/连续谱的展开方式、系数求法、概率含义。
- **代数法（算符法）**：逐步展开——定义算符 → 对易子 → 升降性质 → 基态条件 → 递推构造，用"步骤 N"标注。
- **幂级数法**（如氢原子径向方程）：渐近分析 → 剥离渐近行为 → 幂级数展开 → 递推关系 → 级数截断 → 量子化。此模式在多个问题中反复出现。
- **矩阵表示**（如自旋）：先从升降算符构造矩阵元，再组装为完整矩阵。强调基底选择的约定。
- **角动量合成**：从最高权态出发，反复作用降算符，利用正交性构造下一组态。

### 编程题规范
- 提供完整可运行的 Python 代码框架，注释用中文，绘图为英文。
- 包含参数设置、数值求解、可视化三部分，使用无量纲化处理。

### 文件操作
- 新节追加到已有章节文件（不创建新文件），写入前**重新读取文件**确认最新状态。
- 撰写新节时须阅读已完成小节的风格（标题层级、公式标注、习题编号），严格延续。
